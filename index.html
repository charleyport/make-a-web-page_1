In Unit 1 we learned that the main type of document on the web is HTML. We also learned that web files can be plain text, HTML, images, videos, and music.

HTML-stands for Hyper Text Markup Language. It utilizes -text content, -markup, references to other documents, and -links to other pages

Basic HTML Vocabulary-consists of

o A Tag- An HTML tag is always contained within angled brackets. Most tags have an opening tag and closing tag. Some tags (called "void"tags) do not require a closing tag

o An Element- An HTML element consists of the content contain within the opening and closing tag.

o An Attribute- is a property of an HTML element.

I have learned the so far there are two types of elements:

Inline and Block

Examples of Inline are: "b", "em", and "img". An example of a block element is "p". A block element encapsulates the content into an "invisible box". "Span" and "Div" are also examples of inline and block elements. I have much more to learn as I cannot yet use the left and right arrow in my text without the code trying to create a new element.

An example of a tag that uses an HTML Attribute woulds be the "a" or anchor tag. It would contain an attribute, followed by a "value", the content, and a closing tag.

Moving further into :Webpages,Documents,and Structure,I have learned how the HTML tag and DOM or "Document Object Model". create the elements that form a tree-like structure while CSS controls the style elements of a page.

CCS fonts and formats have a specific hierarchy. The analogy used was in regard to laws.
  • 1.National Laws-govern the entire country (document)
  • 2.State Laws drill down into more specific spots (boxes perhaps)
  • 3.Municipal laws drill down even further to cover even more specifics

Keeping this hierarchy in mind allows a programmer to avoid repetition and long drawn out code through the reuse of "class rules" for boxes that fit the same format and style.

So National law rules unless something more specific is defined

Avoiding repitition or Dry Code(don't repeat yourself)

As defined in The Pragmatic Programmer“Every piece of knowledge must have a single, unambiguous, authoritative representation within a system.”

Looking at my code for this project I see what may be an example of what DRY code is referring to. I believe an example of this is that I share a "div class="summary" in some of my HTML. While this can be benificial in reducing the overall complexity of the code, it also makes it impossible for me to modify this "class" on on line of code without affecting something that I may not want it to change somewhere else in my HTML. The more complex the project, the more likely it is for this to cause issues.
